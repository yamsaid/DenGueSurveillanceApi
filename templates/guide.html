Guide API - Surveillance Dengue

Documentation complète pour l'utilisation de l'API de surveillance sanitaire

Introduction

Cette API permet de consulter, ajouter et exporter des données liées aux cas de dengue au Burkina Faso. Elle s'adresse aux professionnels de santé, chercheurs et développeurs d'applications.

Ajout de Données

Enregistrez facilement de nouveaux cas et patients avec des endpoints REST simples.

Consultation

Récupérez les données et statistiques avec des requêtes flexibles et performantes.

Export

Exportez vos données en CSV, Excel ou JSON pour analyse externe.

Sécurité

Authentification par clé API et validation stricte des données.

Conseil : Utilisez les outils interactifs Swagger UI et ReDoc pour explorer l'API de manière visuelle.

Exemples d'Utilisation

Test Interactif - Ajout d'un Patient

JavaScript 
// Exemple d'ajout d'un patient
const response = await fetch('/api/patients/', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    nom: 'Kaboré',
    age: 35,
    sexe: 'M'
  })
});

const data = await response.json();
console.log('Patient ajouté:', data);
Python 
import requests

# Exemple d'ajout d'un patient
url = 'http://localhost:8000/api/patients/'
headers = {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
}
data = {
    'nom': 'Kaboré',
    'age': 35,
    'sexe': 'M'
}

response = requests.post(url, json=data, headers=headers)
print('Patient ajouté:', response.json())
cURL 
curl -X POST "http://localhost:8000/api/patients/" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "nom": "Kaboré",
    "age": 35,
    "sexe": "M"
  }'

Modèles de Données

Modèle CasSante

Champ Type Obligatoire Description
id int Oui Identifiant unique du cas
date date Oui Date de déclaration du cas
patient_id int Oui Référence au patient
centre_id int Oui Référence au centre médical
symptomes string Non Description des symptômes

Modèle Patient

Champ Type Obligatoire Description
id int Oui Identifiant unique du patient
nom string Oui Nom complet du patient
age int Oui Âge du patient
sexe string Oui Sexe (M/F)
region string Non Région de résidence

Gestion des Erreurs

Note : Toutes les erreurs retournent un objet JSON avec les détails de l'erreur.
Code Type Description Solution
400 Bad Request Requête malformée ou paramètres invalides Vérifiez le format de votre requête
401 Unauthorized Clé API manquante ou invalide Vérifiez votre clé API
404 Not Found Ressource non trouvée Vérifiez l'URL de l'endpoint
422 Unprocessable Entity Erreur de validation des données Vérifiez le format des données
500 Internal Server Error Erreur serveur interne Contactez l'équipe technique
Exemple d'Erreur 
{
  "error": "Validation Error",
  "message": "Les données fournies sont invalides",
  "details": {
    "age": "L'âge doit être un nombre positif",
    "sexe": "Le sexe doit être 'M' ou 'F'"
  },
  "status_code": 422
}
